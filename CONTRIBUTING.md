# Contributing

Thanks for helping improve this project.

## Development

1. Keep changes small and reviewable.
2. Add or update tests for behavior changes.
3. Keep examples sanitized.
4. Run the public-safety check before opening a pull request.

```sh
./scripts/verify-public-safety.sh
```

## Privacy Requirements

Pull requests must not include secrets, private Codex Desktop state, local
runtime configuration, personal logs, or private repository history.
